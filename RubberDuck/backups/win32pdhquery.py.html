1    """ 
2    Performance Data Helper (PDH) Query Classes 
3     
4    Wrapper classes for end-users and high-level access to the PDH query 
5    mechanisms.  PDH is a win32-specific mechanism for accessing the 
6    performance data made available by the system.  The Python for Windows 
7    PDH module does not implement the "Registry" interface, implementing 
8    the more straightforward Query-based mechanism. 
9     
10   The basic idea of a PDH Query is an object which can query the system 
11   about the status of any number of "counters."  The counters are paths 
12   to a particular piece of performance data.  For instance, the path  
13   '\\Memory\\Available Bytes' describes just about exactly what it says 
14   it does, the amount of free memory on the default computer expressed  
15   in Bytes.  These paths can be considerably more complex than this,  
16   but part of the point of this wrapper module is to hide that 
17   complexity from the end-user/programmer. 
18    
19   EXAMPLE: A more complex Path 
20       '\\\\RAISTLIN\\PhysicalDisk(_Total)\\Avg. Disk Bytes/Read' 
21       Raistlin --> Computer Name 
22       PhysicalDisk --> Object Name 
23       _Total --> The particular Instance (in this case, all instances, i.e. all drives) 
24       Avg. Disk Bytes/Read --> The piece of data being monitored. 
25    
26   EXAMPLE: Collecting Data with a Query 
27       As an example, the following code implements a logger which allows the 
28       user to choose what counters they would like to log, and logs those 
29       counters for 30 seconds, at two-second intervals. 
30        
31       query = Query() 
32       query.addcounterbybrowsing() 
33       query.collectdatafor(30,2) 
34        
35       The data is now stored in a list of lists as: 
36       query.curresults 
37        
38       The counters(paths) which were used to collect the data are: 
39       query.curpaths 
40        
41       You can use the win32pdh.ParseCounterPath(path) utility function 
42       to turn the paths into more easily read values for your task, or 
43       write the data to a file, or do whatever you want with it. 
44    
45   OTHER NOTABLE METHODS: 
46       query.collectdatawhile(period) # start a logging thread for collecting data 
47       query.collectdatawhile_stop() # signal the logging thread to stop logging 
48       query.collectdata() # run the query only once 
49       query.addperfcounter(object, counter, machine=None) # add a standard performance counter 
50       query.addinstcounter(object, counter,machine=None,objtype = 'Process',volatile=1,format = win32pdh.PDH_FMT_LONG) # add a possibly volatile counter 
51    
52   ### Known bugs and limitations ### 
53   Due to a problem with threading under the PythonWin interpreter, there 
54   will be no data logged if the PythonWin window is not the foreground 
55   application.  Workaround: scripts using threading should be run in the 
56   python.exe interpreter. 
57    
58   The volatile-counter handlers are possibly buggy, they haven't been 
59   tested to any extent.  The wrapper Query makes it safe to pass invalid 
60   paths (a -1 will be returned, or the Query will be totally ignored, 
61   depending on the missing element), so you should be able to work around 
62   the error by including all possible paths and filtering out the -1's. 
63    
64   There is no way I know of to stop a thread which is currently sleeping, 
65   so you have to wait until the thread in collectdatawhile is activated 
66   again.  This might become a problem in situations where the collection 
67   period is multiple minutes (or hours, or whatever). 
68    
69   Should make the win32pdh.ParseCounter function available to the Query 
70   classes as a method or something similar, so that it can be accessed 
71   by programmes that have just picked up an instance from somewhere. 
72    
73   Should explicitly mention where QueryErrors can be raised, and create a 
74   full test set to see if there are any uncaught win32api.error's still 
75   hanging around. 
76    
77   When using the python.exe interpreter, the addcounterbybrowsing- 
78   generated browser window is often hidden behind other windows.  No known 
79   workaround other than Alt-tabing to reach the browser window. 
80

131  class BaseQuery:
132      """ 
133      Provides wrapped access to the Performance Data Helper query 
134      objects, generally you should use the child class Query 
135      unless you have need of doing weird things :) 
136   
137      This class supports two major working paradigms.  In the first, 
138      you open the query, and run it as many times as you need, closing 
139      the query when you're done with it.  This is suitable for static 
140      queries (ones where processes being monitored don't disappear). 
141   
142      In the second, you allow the query to be opened each time and 
143      closed afterward.  This causes the base query object to be 
144      destroyed after each call.  Suitable for dynamic queries (ones 
145      which watch processes which might be closed while watching.) 
146      """
147  
148      def __init__(self, paths=None):
149          """ 
150          The PDH Query object is initialised with a single, optional 
151          list argument, that must be properly formatted PDH Counter 
152          paths.  Generally this list will only be provided by the class 
153          when it is being unpickled (removed from storage).  Normal 
154          use is to call the class with no arguments and use the various 
155          addcounter functions (particularly, for end user's, the use of 
156          addcounterbybrowsing is the most common approach)  You might 
157          want to provide the list directly if you want to hard-code the 
158          elements with which your query deals (and thereby avoid the 
159          overhead of unpickling the class). 
160          """
161          self.counters = []
162          if paths:
163              self.paths = paths
164          else:
165              self.paths = []
166          self._base = None
167          self.active = 0
168          self.curpaths = []
169  
170      def addcounterbybrowsing(
171          self, flags=win32pdh.PERF_DETAIL_WIZARD, windowtitle="Python Browser"
172      ):
173          """ 
174          Adds possibly multiple paths to the paths attribute of the query, 
175          does this by calling the standard counter browsing dialogue.  Within 
176          this dialogue, find the counter you want to log, and click: Add, 
177          repeat for every path you want to log, then click on close.  The 
178          paths are appended to the non-volatile paths list for this class, 
179          subclasses may create a function which parses the paths and decides 
180          (via heuristics) whether to add the path to the volatile or non-volatile 
181          path list. 
182          e.g.: 
183                  query.addcounter() 
184          """
185          win32pdh.BrowseCounters(None, 0, self.paths.append, flags, windowtitle)
186  
187      def rawaddcounter(self, object, counter, instance=None, inum=-1, machine=None):
188          """ 
189          Adds a single counter path, without catching any exceptions. 
190   
191          See addcounter for details. 
192          """
193          path = win32pdh.MakeCounterPath(
194              (machine, object, instance, None, inum, counter)
195          )
196          self.paths.append(path)
197  
198      def addcounter(self, object, counter, instance=None, inum=-1, machine=None):
199          """ 
200          Adds a single counter path to the paths attribute.  Normally 
201          this will be called by a child class' speciality functions, 
202          rather than being called directly by the user. (Though it isn't 
203          hard to call manually, since almost everything is given a default) 
204          This method is only functional when the query is closed (or hasn't 
205          yet been opened).  This is to prevent conflict in multi-threaded 
206          query applications). 
207          e.g.: 
208                  query.addcounter('Memory','Available Bytes') 
209          """
210          if not self.active:
211              try:
212                  self.rawaddcounter(object, counter, instance, inum, machine)
213                  return 0
214              except win32api.error:
215                  return -1
216          else:
217              return -1
218  
219      def open(self):
220          """ 
221          Build the base query object for this wrapper, 
222          then add all of the counters required for the query. 
223          Raise a QueryError if we can't complete the functions. 
224          If we are already open, then do nothing. 
225          """
226          if not self.active:  # to prevent having multiple open queries
227              # curpaths are made accessible here because of the possibility of volatile paths
228              # which may be dynamically altered by subclasses.
229              self.curpaths = copy.copy(self.paths)
230              try:
231                  base = win32pdh.OpenQuery()
232                  for path in self.paths:
233                      try:
234                          self.counters.append(win32pdh.AddCounter(base, path))
235                      except win32api.error:  # we passed a bad path
236                          self.counters.append(0)
237                          pass
238                  self._base = base
239                  self.active = 1
240                  return 0  # open succeeded
241              except:  # if we encounter any errors, kill the Query
242                  try:
243                      self.killbase(base)
244                  except NameError:  # failed in creating query
245                      pass
246                  self.active = 0
247                  self.curpaths = []
248                  raise QueryError(self)
249          return 1  # already open
250  
251      def killbase(self, base=None):
252          """ 
253          ### This is not a public method 
254          Mission critical function to kill the win32pdh objects held 
255          by this object.  User's should generally use the close method 
256          instead of this method, in case a sub-class has overridden 
257          close to provide some special functionality. 
258          """
259          # Kill Pythonic references to the objects in this object's namespace
260          self._base = None
261          counters = self.counters
262          self.counters = []
263          # we don't kill the curpaths for convenience, this allows the
264          # user to close a query and still access the last paths
265          self.active = 0
266          # Now call the delete functions on all of the objects
267          try:
268              map(win32pdh.RemoveCounter, counters)
269          except:
270              pass
271          try:
272              win32pdh.CloseQuery(base)
273          except:
274              pass
275          del counters
276          del base
277  
278      def close(self):
279          """ 
280          Makes certain that the underlying query object has been closed, 
281          and that all counters have been removed from it.  This is 
282          important for reference counting. 
283          You should only need to call close if you have previously called 
284          open.  The collectdata methods all can handle opening and 
285          closing the query.  Calling close multiple times is acceptable. 
286          """
287          try:
288              self.killbase(self._base)
289          except AttributeError:
290              self.killbase()
291  
292      __del__ = close
293  
294      def collectdata(self, format=win32pdh.PDH_FMT_LONG):
295          """ 
296          Returns the formatted current values for the Query 
297          """
298          if self._base:  # we are currently open, don't change this
299              return self.collectdataslave(format)
300          else:  # need to open and then close the _base, should be used by one-offs and elements tracking application instances
301              self.open()  # will raise QueryError if couldn't open the query
302              temp = self.collectdataslave(format)
303              self.close()  # will always close
304              return temp
305  
306      def collectdataslave(self, format=win32pdh.PDH_FMT_LONG):
307          """ 
308          ### Not a public method 
309          Called only when the Query is known to be open, runs over 
310          the whole set of counters, appending results to the temp, 
311          returns the values as a list. 
312          """
313          try:
314              win32pdh.CollectQueryData(self._base)
315              temp = []
316              for counter in self.counters:
317                  ok = 0
318                  try:
319                      if counter:
320                          temp.append(
321                              win32pdh.GetFormattedCounterValue(counter, format)[1]
322                          )
323                          ok = 1
324                  except win32api.error:
325                      pass
326                  if not ok:
327                      temp.append(-1)  # a better way to signal failure???
328              return temp
329          except win32api.error:  # will happen if, for instance, no counters are part of the query and we attempt to collect data for it.
330              return [-1] * len(self.counters)
331  
332      # pickle functions
333      def __getinitargs__(self):
334          """ 
335          ### Not a public method 
336          """
337          return (self.paths,)

340  class Query(BaseQuery):
341      """ 
342      Performance Data Helper(PDH) Query object: 
343   
344      Provides a wrapper around the native PDH query object which 
345      allows for query reuse, query storage, and general maintenance 
346      functions (adding counter paths in various ways being the most 
347      obvious ones). 
348      """
349  
350      def __init__(self, *args, **namedargs):
351          """ 
352          The PDH Query object is initialised with a single, optional 
353          list argument, that must be properly formatted PDH Counter 
354          paths.  Generally this list will only be provided by the class 
355          when it is being unpickled (removed from storage).  Normal 
356          use is to call the class with no arguments and use the various 
357          addcounter functions (particularly, for end user's, the use of 
358          addcounterbybrowsing is the most common approach)  You might 
359          want to provide the list directly if you want to hard-code the 
360          elements with which your query deals (and thereby avoid the 
361          overhead of unpickling the class). 
362          """
363          self.volatilecounters = []
364          BaseQuery.__init__(*(self,) + args, **namedargs)
365  
366      def addperfcounter(self, object, counter, machine=None):
367          """ 
368          A "Performance Counter" is a stable, known, common counter, 
369          such as Memory, or Processor.  The use of addperfcounter by 
370          end-users is deprecated, since the use of 
371          addcounterbybrowsing is considerably more flexible and general. 
372          It is provided here to allow the easy development of scripts 
373          which need to access variables so common we know them by name 
374          (such as Memory|Available Bytes), and to provide symmetry with 
375          the add inst counter method. 
376          usage: 
377                  query.addperfcounter('Memory', 'Available Bytes') 
378          It is just as easy to access addcounter directly, the following 
379          has an identicle effect. 
380                  query.addcounter('Memory', 'Available Bytes') 
381          """
382          BaseQuery.addcounter(self, object=object, counter=counter, machine=machine)
383  
384      def addinstcounter(
385          self,
386          object,
387          counter,
388          machine=None,
389          objtype="Process",
390          volatile=1,
391          format=win32pdh.PDH_FMT_LONG,
392      ):
393          """ 
394          The purpose of using an instcounter is to track particular 
395          instances of a counter object (e.g. a single processor, a single 
396          running copy of a process).  For instance, to track all python.exe 
397          instances, you would need merely to ask: 
398                  query.addinstcounter('python','Virtual Bytes') 
399          You can find the names of the objects and their available counters 
400          by doing an addcounterbybrowsing() call on a query object (or by 
401          looking in performance monitor's add dialog.) 
402   
403          Beyond merely rearranging the call arguments to make more sense, 
404          if the volatile flag is true, the instcounters also recalculate 
405          the paths of the available instances on every call to open the 
406          query. 
407          """
408          if volatile:
409              self.volatilecounters.append((object, counter, machine, objtype, format))
410          else:
411              self.paths[len(self.paths) :] = self.getinstpaths(
412                  object, counter, machine, objtype, format
413              )
414  
415      def getinstpaths(
416          self,
417          object,
418          counter,
419          machine=None,
420          objtype="Process",
421          format=win32pdh.PDH_FMT_LONG,
422      ):
423          """ 
424          ### Not an end-user function 
425          Calculate the paths for an instance object. Should alter 
426          to allow processing for lists of object-counter pairs. 
427          """
428          items, instances = win32pdh.EnumObjectItems(None, None, objtype, -1)
429          # find out how many instances of this element we have...
430          instances.sort()
431          try:
432              cur = instances.index(object)
433          except ValueError:
434              return []  # no instances of this object
435          temp = [object]
436          try:
437              while instances[cur + 1] == object:
438                  temp.append(object)
439                  cur = cur + 1
440          except IndexError:  # if we went over the end
441              pass
442          paths = []
443          for ind in range(len(temp)):
444              # can this raise an error?
445              paths.append(
446                  win32pdh.MakeCounterPath(
447                      (machine, "Process", object, None, ind, counter)
448                  )
449              )
450          return paths  # should also return the number of elements for naming purposes
451  
452      def open(self, *args, **namedargs):
453          """ 
454          Explicitly open a query: 
455          When you are needing to make multiple calls to the same query, 
456          it is most efficient to open the query, run all of the calls, 
457          then close the query, instead of having the collectdata method 
458          automatically open and close the query each time it runs. 
459          There are currently no arguments to open. 
460          """
461          # do all the normal opening stuff, self._base is now the query object
462          BaseQuery.open(*(self,) + args, **namedargs)
463          # should rewrite getinstpaths to take a single tuple
464          paths = []
465          for tup in self.volatilecounters:
466              paths[len(paths) :] = self.getinstpaths(*tup)
467          for path in paths:
468              try:
469                  self.counters.append(win32pdh.AddCounter(self._base, path))
470                  self.curpaths.append(
471                      path
472                  )  # if we fail on the line above, this path won't be in the table or the counters
473              except win32api.error:
474                  pass  # again, what to do with a malformed path???
475  
476      def collectdatafor(self, totalperiod, period=1):
477          """ 
478          Non-threaded collection of performance data: 
479          This method allows you to specify the total period for which you would 
480          like to run the Query, and the time interval between individual 
481          runs.  The collected data is stored in query.curresults at the 
482          _end_ of the run.  The pathnames for the query are stored in 
483          query.curpaths. 
484          e.g.: 
485                  query.collectdatafor(30,2) 
486          Will collect data for 30seconds at 2 second intervals 
487          """
488          tempresults = []
489          try:
490              self.open()
491              for ind in range(totalperiod / period):
492                  tempresults.append(self.collectdata())
493                  time.sleep(period)
494              self.curresults = tempresults
495          finally:
496              self.close()
497  
498      def collectdatawhile(self, period=1):
499          """ 
500          Threaded collection of performance data: 
501          This method sets up a simple semaphor system for signalling 
502          when you would like to start and stop a threaded data collection 
503          method.  The collection runs every period seconds until the 
504          semaphor attribute is set to a non-true value (which normally 
505          should be done by calling query.collectdatawhile_stop() .) 
506          e.g.: 
507                  query.collectdatawhile(2) 
508                  # starts the query running, returns control to the caller immediately 
509                  # is collecting data every two seconds. 
510                  # do whatever you want to do while the thread runs, then call: 
511                  query.collectdatawhile_stop() 
512                  # when you want to deal with the data.  It is generally a good idea 
513                  # to sleep for period seconds yourself, since the query will not copy 
514                  # the required data until the next iteration: 
515                  time.sleep(2) 
516                  # now you can access the data from the attributes of the query 
517                  query.curresults 
518                  query.curpaths 
519          """
520          self.collectdatawhile_active = 1
521          _thread.start_new_thread(self.collectdatawhile_slave, (period,))
522  
523      def collectdatawhile_stop(self):
524          """ 
525          Signals the collectdatawhile slave thread to stop collecting data 
526          on the next logging iteration. 
527          """
528          self.collectdatawhile_active = 0
529  
530      def collectdatawhile_slave(self, period):
531          """ 
532          ### Not a public function 
533          Does the threaded work of collecting the data and storing it 
534          in an attribute of the class. 
535          """
536          tempresults = []
537          try:
538              self.open()  # also sets active, so can't be changed.
539              while self.collectdatawhile_active:
540                  tempresults.append(self.collectdata())
541                  time.sleep(period)
542              self.curresults = tempresults
543          finally:
544              self.close()
545
