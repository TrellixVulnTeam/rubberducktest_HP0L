relativelayout.py 
1    ''' 
2    Relative Layout 
3    =============== 
4     
5    .. versionadded:: 1.4.0 
6     
7     
8    This layout allows you to set relative coordinates for children. If you want 
9    absolute positioning, use the :class:`~kivy.uix.floatlayout.FloatLayout`. 
10    
11   The :class:`RelativeLayout` class behaves just like the regular 
12   :class:`FloatLayout` except that its child widgets are positioned relative to 
13   the layout. 
14    
15   When a widget with position = (0,0) is added to a RelativeLayout, 
16   the child widget will also move when the position of the RelativeLayout 
17   is changed. The child widgets coordinates remain (0,0) as they are 
18   always relative to the parent layout. 
19    
20   Coordinate Systems 
21   ------------------ 
22    
23   Window coordinates 
24   ~~~~~~~~~~~~~~~~~~ 
25    
26   By default, there's only one coordinate system that defines the position of 
27   widgets and touch events dispatched to them: the window coordinate system, 
28   which places (0, 0) at the bottom left corner of the window. 
29   Although there are other coordinate systems defined, e.g. local 
30   and parent coordinates, these coordinate systems are identical to the window 
31   coordinate system as long as a relative layout type widget is not in the 
32   widget's parent stack. When widget.pos is read or a touch is received, 
33   the coordinate values are in parent coordinates. But as mentioned, these are 
34   identical to window coordinates, even in complex widget stacks as long as 
35   there's no relative layout type widget in the widget's parent stack. 
36    
37   For example: 
38    
39   .. code-block:: kv 
40    
41       BoxLayout: 
42           Label: 
43               text: 'Left' 
44           Button: 
45               text: 'Middle' 
46               on_touch_down: print('Middle: {}'.format(args[1].pos)) 
47           BoxLayout: 
48               on_touch_down: print('Box: {}'.format(args[1].pos)) 
49               Button: 
50                   text: 'Right' 
51                   on_touch_down: print('Right: {}'.format(args[1].pos)) 
52    
53   When the middle button is clicked and the touch propagates through the 
54   different parent coordinate systems, it prints the following:: 
55    
56       >>> Box: (430.0, 282.0) 
57       >>> Right: (430.0, 282.0) 
58       >>> Middle: (430.0, 282.0) 
59    
60   As claimed, the touch has identical coordinates to the window coordinates 
61   in every coordinate system. :meth:`~kivy.uix.widget.Widget.collide_point` 
62   for example, takes the point in window coordinates. 
63    
64   Parent coordinates 
65   ~~~~~~~~~~~~~~~~~~ 
66    
67   Other :class:`RelativeLayout` type widgets are 
68   :class:`~kivy.uix.scatter.Scatter`, 
69   :class:`~kivy.uix.scatterlayout.ScatterLayout`, 
70   and :class:`~kivy.uix.scrollview.ScrollView`. If such a special widget is in 
71   the parent stack, only then does the parent and local coordinate system 
72   diverge from the window coordinate system. For each such widget in the stack, 
73   a coordinate system with (0, 0) of that coordinate system being at the bottom 
74   left corner of that widget is created. **Position and touch coordinates 
75   received and read by a widget are in the coordinate system of the most 
76   recent special widget in its parent stack (not including itself) or in window 
77   coordinates if there are none** (as in the first example). We call these 
78   coordinates parent coordinates. 
79    
80    
81   For example: 
82    
83   .. code-block:: kv 
84    
85       BoxLayout: 
86           Label: 
87               text: 'Left' 
88           Button: 
89               text: 'Middle' 
90               on_touch_down: print('Middle: {}'.format(args[1].pos)) 
91           RelativeLayout: 
92               on_touch_down: print('Relative: {}'.format(args[1].pos)) 
93               Button: 
94                   text: 'Right' 
95                   on_touch_down: print('Right: {}'.format(args[1].pos)) 
96    
97   Clicking on the middle button prints:: 
98    
99       >>> Relative: (396.0, 298.0) 
100      >>> Right: (-137.33, 298.0) 
101      >>> Middle: (396.0, 298.0) 
102   
103  As the touch propagates through the widgets, for each widget, the 
104  touch is received in parent coordinates. Because both the relative and middle 
105  widgets don't have these special widgets in their parent stack, the touch is 
106  the same as window coordinates. Only the right widget, which has a 
107  RelativeLayout in its parent stack, receives the touch in coordinates relative 
108  to that RelativeLayout which is different than window coordinates. 
109   
110  Local and Widget coordinates 
111  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~ 
112   
113  When expressed in parent coordinates, the position is expressed in the 
114  coordinates of the most recent special widget in its parent stack, not 
115  including itself. When expressed in local or widget coordinates, the widgets 
116  themselves are also included. 
117   
118  Changing the above example to transform the parent coordinates into local 
119  coordinates: 
120   
121  .. code-block:: kv 
122   
123      BoxLayout: 
124          Label: 
125              text: 'Left' 
126          Button: 
127              text: 'Middle' 
128              on_touch_down: print('Middle: {}'.format(\ 
129  self.to_local(*args[1].pos))) 
130          RelativeLayout: 
131              on_touch_down: print('Relative: {}'.format(\ 
132  self.to_local(*args[1].pos))) 
133              Button: 
134                  text: 'Right' 
135                  on_touch_down: print('Right: {}'.format(\ 
136  self.to_local(*args[1].pos))) 
137   
138  Now, clicking on the middle button prints:: 
139   
140      >>> Relative: (-135.33, 301.0) 
141      >>> Right: (-135.33, 301.0) 
142      >>> Middle: (398.0, 301.0) 
143   
144  This is because now the relative widget also expresses the coordinates 
145  relative to itself. 
146   
147  .. note:: 
148   
149      Although all widgets including :class:`RelativeLayout` receive their touch 
150      events in ``on_touch_xxx`` in parent coordinates, these special widgets 
151      will transform the touch position to be in local coordinates before it 
152      calls ``super``. This may only be noticeable in a complex inheritance 
153      class. 
154   
155  Coordinate transformations 
156  ~~~~~~~~~~~~~~~~~~~~~~~~~~ 
157   
158  :class:`~kivy.uix.widget.Widget` provides 4 functions to transform coordinates 
159  between the various coordinate systems. For now, we assume that the `relative` 
160  keyword of these functions is `False`. 
161  :meth:`~kivy.uix.widget.Widget.to_widget` takes the coordinates expressed in 
162  window coordinates and returns them in local (widget) coordinates. 
163  :meth:`~kivy.uix.widget.Widget.to_window` takes the coordinates expressed in 
164  local coordinates and returns them in window coordinates. 
165  :meth:`~kivy.uix.widget.Widget.to_parent` takes the coordinates expressed in 
166  local coordinates and returns them in parent coordinates. 
167  :meth:`~kivy.uix.widget.Widget.to_local` takes the coordinates expressed in 
168  parent coordinates and returns them in local coordinates. 
169   
170  Each of the 4 transformation functions take a `relative` parameter. When the 
171  relative parameter is True, the coordinates are returned or originate in 
172  true relative coordinates - relative to a coordinate system with its (0, 0) at 
173  the bottom left corner of the widget in question. 
174   
175  .. _kivy-uix-relativelayout-common-pitfalls: 
176   
177  Common Pitfalls 
178  --------------- 
179   
180  As all positions within a :class:`RelativeLayout` are relative to the position 
181  of the layout itself, the position of the layout should never be used in 
182  determining the position of sub-widgets or the layout's :attr:`canvas`. 
183   
184  Take the following kv code for example: 
185   
186  .. container:: align-right 
187   
188      .. figure:: images/relativelayout-fixedposition.png 
189          :scale: 50% 
190   
191          expected result 
192   
193      .. figure:: images/relativelayout-doubleposition.png 
194          :scale: 50% 
195   
196          actual result 
197   
198  .. code-block:: kv 
199   
200      FloatLayout: 
201          Widget: 
202              size_hint: None, None 
203              size: 200, 200 
204              pos: 200, 200 
205   
206              canvas: 
207                  Color: 
208                      rgba: 1, 1, 1, 1 
209                  Rectangle: 
210                      pos: self.pos 
211                      size: self.size 
212   
213          RelativeLayout: 
214              size_hint: None, None 
215              size: 200, 200 
216              pos: 200, 200 
217   
218              canvas: 
219                  Color: 
220                      rgba: 1, 0, 0, 0.5 
221                  Rectangle: 
222                      pos: self.pos  # incorrect 
223                      size: self.size 
224   
225  You might expect this to render a single pink rectangle; however, the content 
226  of the :class:`RelativeLayout` is already transformed, so the use of 
227  `pos: self.pos` will double that transformation. In this case, using 
228  `pos: 0, 0` or omitting `pos` completely will provide the expected result. 
229   
230  This also applies to the position of sub-widgets. Instead of positioning a 
231  :class:`~kivy.uix.widget.Widget` based on the layout's own position: 
232   
233  .. code-block:: kv 
234   
235      RelativeLayout: 
236          Widget: 
237              pos: self.parent.pos 
238          Widget: 
239              center: self.parent.center 
240   
241  use the :attr:`pos_hint` property: 
242   
243  .. code-block:: kv 
244   
245      RelativeLayout: 
246          Widget: 
247              pos_hint: {'x': 0, 'y': 0} 
248          Widget: 
249              pos_hint: {'center_x': 0.5, 'center_y': 0.5} 
250   
251  .. versionchanged:: 1.7.0 
252      Prior to version 1.7.0, the :class:`RelativeLayout` was implemented as a 
253      :class:`~kivy.uix.floatlayout.FloatLayout` inside a 
254      :class:`~kivy.uix.scatter.Scatter`. This behaviour/widget has 
255      been renamed to `ScatterLayout`. The :class:`RelativeLayout` now only 
256      supports relative positions (and can't be rotated, scaled or translated on 
257      a multitouch system using two or more fingers). This was done so that the 
258      implementation could be optimized and avoid the heavier calculations of 
259      :class:`Scatter` (e.g. inverse matrix, recalculating multiple properties 
260      etc.) 
261   
262  '''

266  from kivy.uix.floatlayout import FloatLayout

269  class RelativeLayout(FloatLayout):
270      '''RelativeLayout class, see module documentation for more information. 
271      '''
272  
273      def __init__(self, **kw):
274          super(RelativeLayout, self).__init__(**kw)
275          funbind = self.funbind
276          trigger = self._trigger_layout
277          funbind('pos', trigger)
278          funbind('pos_hint', trigger)
279  
280      def do_layout(self, *args):
281          super(RelativeLayout, self).do_layout(pos=(0, 0))
282  
283      def to_parent(self, x, y, **k):
284          return (x + self.x, y + self.y)
285  
286      def to_local(self, x, y, **k):
287          return (x - self.x, y - self.y)
288  
289      def _apply_transform(self, m, pos=None):
290          m.translate(self.x, self.y, 0)
291          return super(RelativeLayout, self)._apply_transform(m, (0, 0))
292  
293      def on_touch_down(self, touch):
294          x, y = touch.x, touch.y
295          touch.push()
296          touch.apply_transform_2d(self.to_local)
297          ret = super(RelativeLayout, self).on_touch_down(touch)
298          touch.pop()
299          return ret
300  
301      def on_touch_move(self, touch):
302          x, y = touch.x, touch.y
303          touch.push()
304          touch.apply_transform_2d(self.to_local)
305          ret = super(RelativeLayout, self).on_touch_move(touch)
306          touch.pop()
307          return ret
308  
309      def on_touch_up(self, touch):
310          x, y = touch.x, touch.y
311          touch.push()
312          touch.apply_transform_2d(self.to_local)
313          ret = super(RelativeLayout, self).on_touch_up(touch)
314          touch.pop()
315          return ret
