screenmanager.py 
1    '''Screen Manager 
2    ============== 
3     
4    .. image:: images/screenmanager.gif 
5        :align: right 
6     
7    .. versionadded:: 1.4.0 
8     
9    The screen manager is a widget dedicated to managing multiple screens for your 
10   application. The default :class:`ScreenManager` displays only one 
11   :class:`Screen` at a time and uses a :class:`TransitionBase` to switch from one 
12   Screen to another. 
13    
14   Multiple transitions are supported based on changing the screen coordinates / 
15   scale or even performing fancy animation using custom shaders. 
16    
17   Basic Usage 
18   ----------- 
19    
20   Let's construct a Screen Manager with 4 named screens. When you are creating 
21   a screen, **you absolutely need to give a name to it**:: 
22    
23       from kivy.uix.screenmanager import ScreenManager, Screen 
24    
25       # Create the manager 
26       sm = ScreenManager() 
27    
28       # Add few screens 
29       for i in range(4): 
30           screen = Screen(name='Title %d' % i) 
31           sm.add_widget(screen) 
32    
33       # By default, the first screen added into the ScreenManager will be 
34       # displayed. You can then change to another screen. 
35    
36       # Let's display the screen named 'Title 2' 
37       # A transition will automatically be used. 
38       sm.current = 'Title 2' 
39    
40   The default :attr:`ScreenManager.transition` is a :class:`SlideTransition` with 
41   options :attr:`~SlideTransition.direction` and 
42   :attr:`~TransitionBase.duration`. 
43    
44   Please note that by default, a :class:`Screen` displays nothing: it's just a 
45   :class:`~kivy.uix.relativelayout.RelativeLayout`. You need to use that class as 
46   a root widget for your own screen, the best way being to subclass. 
47    
48   .. warning:: 
49       As :class:`Screen` is a :class:`~kivy.uix.relativelayout.RelativeLayout`, 
50       it is important to understand the 
51       :ref:`kivy-uix-relativelayout-common-pitfalls`. 
52    
53   Here is an example with a 'Menu Screen' and a 'Settings Screen':: 
54    
55       from kivy.app import App 
56       from kivy.lang import Builder 
57       from kivy.uix.screenmanager import ScreenManager, Screen 
58    
59       # Create both screens. Please note the root.manager.current: this is how 
60       # you can control the ScreenManager from kv. Each screen has by default a 
61       # property manager that gives you the instance of the ScreenManager used. 
62       Builder.load_string(""" 
63       <MenuScreen>: 
64           BoxLayout: 
65               Button: 
66                   text: 'Goto settings' 
67                   on_press: root.manager.current = 'settings' 
68               Button: 
69                   text: 'Quit' 
70    
71       <SettingsScreen>: 
72           BoxLayout: 
73               Button: 
74                   text: 'My settings button' 
75               Button: 
76                   text: 'Back to menu' 
77                   on_press: root.manager.current = 'menu' 
78       """) 
79    
80       # Declare both screens 
81       class MenuScreen(Screen): 
82           pass 
83    
84       class SettingsScreen(Screen): 
85           pass 
86    
87       class TestApp(App): 
88    
89           def build(self): 
90               # Create the screen manager 
91               sm = ScreenManager() 
92               sm.add_widget(MenuScreen(name='menu')) 
93               sm.add_widget(SettingsScreen(name='settings')) 
94    
95               return sm 
96    
97       if __name__ == '__main__': 
98           TestApp().run() 
99    
100   
101  Changing Direction 
102  ------------------ 
103   
104  A common use case for :class:`ScreenManager` involves using a 
105  :class:`SlideTransition` which slides right to the next screen 
106  and slides left to the previous screen. Building on the previous 
107  example, this can be accomplished like so:: 
108   
109      Builder.load_string(""" 
110      <MenuScreen>: 
111          BoxLayout: 
112              Button: 
113                  text: 'Goto settings' 
114                  on_press: 
115                      root.manager.transition.direction = 'left' 
116                      root.manager.current = 'settings' 
117              Button: 
118                  text: 'Quit' 
119   
120      <SettingsScreen>: 
121          BoxLayout: 
122              Button: 
123                  text: 'My settings button' 
124              Button: 
125                  text: 'Back to menu' 
126                  on_press: 
127                      root.manager.transition.direction = 'right' 
128                      root.manager.current = 'menu' 
129      """) 
130   
131   
132  Advanced Usage 
133  -------------- 
134   
135  From 1.8.0, you can now switch dynamically to a new screen, change the 
136  transition options and remove the previous one by using 
137  :meth:`~ScreenManager.switch_to`:: 
138   
139      sm = ScreenManager() 
140      screens = [Screen(name='Title {}'.format(i)) for i in range(4)] 
141   
142      sm.switch_to(screens[0]) 
143      # later 
144      sm.switch_to(screens[1], direction='right') 
145   
146  Note that this method adds the screen to the :class:`ScreenManager` instance 
147  and should not be used if your screens have already been added to this 
148  instance. To switch to a screen which is already added, you should use the 
149  :attr:`~ScreenManager.current` property. 
150   
151   
152  Changing transitions 
153  -------------------- 
154   
155  You have multiple transitions available by default, such as: 
156   
157  - :class:`NoTransition` - switches screens instantly with no animation 
158  - :class:`SlideTransition` - slide the screen in/out, from any direction 
159  - :class:`CardTransition` - new screen slides on the previous 
160    or the old one slides off the new one depending on the mode 
161  - :class:`SwapTransition` - implementation of the iOS swap transition 
162  - :class:`FadeTransition` - shader to fade the screen in/out 
163  - :class:`WipeTransition` - shader to wipe the screens from right to left 
164  - :class:`FallOutTransition` - shader where the old screen 'falls' and 
165    becomes transparent, revealing the new one behind it. 
166  - :class:`RiseInTransition` - shader where the new screen rises from the 
167    screen centre while fading from transparent to opaque. 
168   
169  You can easily switch transitions by changing the 
170  :attr:`ScreenManager.transition` property:: 
171   
172      sm = ScreenManager(transition=FadeTransition()) 
173   
174  .. note:: 
175   
176      Currently, none of Shader based Transitions use 
177      anti-aliasing. This is because they use the FBO which doesn't have 
178      any logic to handle supersampling. This is a known issue and we 
179      are working on a transparent implementation that will give the 
180      same results as if it had been rendered on screen. 
181   
182      To be more concrete, if you see sharp edged text during the animation, it's 
183      normal. 
184

215  class Screen(RelativeLayout):
216      '''Screen is an element intended to be used with a :class:`ScreenManager`. 
217      Check module documentation for more information. 
218   
219      :Events: 
220          `on_pre_enter`: () 
221              Event fired when the screen is about to be used: the entering 
222              animation is started. 
223          `on_enter`: () 
224              Event fired when the screen is displayed: the entering animation is 
225              complete. 
226          `on_pre_leave`: () 
227              Event fired when the screen is about to be removed: the leaving 
228              animation is started. 
229          `on_leave`: () 
230              Event fired when the screen is removed: the leaving animation is 
231              finished. 
232   
233      .. versionchanged:: 1.6.0 
234          Events `on_pre_enter`, `on_enter`, `on_pre_leave` and `on_leave` were 
235          added. 
236      '''
237  
238      name = StringProperty('')
239      ''' 
240      Name of the screen which must be unique within a :class:`ScreenManager`. 
241      This is the name used for :attr:`ScreenManager.current`. 
242   
243      :attr:`name` is a :class:`~kivy.properties.StringProperty` and defaults to 
244      ''. 
245      '''
246  
247      manager = ObjectProperty(None, allownone=True)
248      ''':class:`ScreenManager` object, set when the screen is added to a 
249      manager. 
250   
251      :attr:`manager` is an :class:`~kivy.properties.ObjectProperty` and 
252      defaults to None, read-only. 
253   
254      '''
255  
256      transition_progress = NumericProperty(0.)
257      '''Value that represents the completion of the current transition, if any 
258      is occurring. 
259   
260      If a transition is in progress, whatever the mode, the value will change 
261      from 0 to 1. If you want to know if it's an entering or leaving animation, 
262      check the :attr:`transition_state`. 
263   
264      :attr:`transition_progress` is a :class:`~kivy.properties.NumericProperty` 
265      and defaults to 0. 
266      '''
267  
268      transition_state = OptionProperty('out', options=('in', 'out'))
269      '''Value that represents the state of the transition: 
270   
271      - 'in' if the transition is going to show your screen 
272      - 'out' if the transition is going to hide your screen 
273   
274      After the transition is complete, the state will retain its last value (in 
275      or out). 
276   
277      :attr:`transition_state` is an :class:`~kivy.properties.OptionProperty` and 
278      defaults to 'out'. 
279      '''
280  
281      __events__ = ('on_pre_enter', 'on_enter', 'on_pre_leave', 'on_leave')
282  
283      def on_pre_enter(self, *args):
284          pass
285  
286      def on_enter(self, *args):
287          pass
288  
289      def on_pre_leave(self, *args):
290          pass
291  
292      def on_leave(self, *args):
293          pass
294  
295      def __repr__(self):
296          return '<Screen name=%r>' % self.name

299  class TransitionBase(EventDispatcher):
300      '''TransitionBase is used to animate 2 screens within the 
301      :class:`ScreenManager`. This class acts as a base for other 
302      implementations like the :class:`SlideTransition` and 
303      :class:`SwapTransition`. 
304   
305      :Events: 
306          `on_progress`: Transition object, progression float 
307              Fired during the animation of the transition. 
308          `on_complete`: Transition object 
309              Fired when the transition is finished. 
310      '''
311  
312      screen_out = ObjectProperty()
313      '''Property that contains the screen to hide. 
314      Automatically set by the :class:`ScreenManager`. 
315   
316      :class:`screen_out` is an :class:`~kivy.properties.ObjectProperty` and 
317      defaults to None. 
318      '''
319  
320      screen_in = ObjectProperty()
321      '''Property that contains the screen to show. 
322      Automatically set by the :class:`ScreenManager`. 
323   
324      :class:`screen_in` is an :class:`~kivy.properties.ObjectProperty` and 
325      defaults to None. 
326      '''
327  
328      duration = NumericProperty(.4)
329      '''Duration in seconds of the transition. 
330   
331      :class:`duration` is a :class:`~kivy.properties.NumericProperty` and 
332      defaults to .4 (= 400ms). 
333   
334      .. versionchanged:: 1.8.0 
335   
336          Default duration has been changed from 700ms to 400ms. 
337      '''
338  
339      manager = ObjectProperty()
340      ''':class:`ScreenManager` object, set when the screen is added to a 
341      manager. 
342   
343      :attr:`manager` is an :class:`~kivy.properties.ObjectProperty` and 
344      defaults to None, read-only. 
345   
346      '''
347  
348      is_active = BooleanProperty(False)
349      '''Indicate whether the transition is currently active or not. 
350   
351      :attr:`is_active` is a :class:`~kivy.properties.BooleanProperty` and 
352      defaults to False, read-only. 
353      '''
354  
355      # privates
356  
357      _anim = ObjectProperty(allownone=True)
358  
359      __events__ = ('on_progress', 'on_complete')
360  
361      def start(self, manager):
362          '''(internal) Starts the transition. This is automatically 
363          called by the :class:`ScreenManager`. 
364          '''
365          if self.is_active:
366              raise ScreenManagerException('start() is called twice!')
367          self.manager = manager
368          self._anim = Animation(d=self.duration, s=0)
369          self._anim.bind(on_progress=self._on_progress,
370                          on_complete=self._on_complete)
371  
372          self.add_screen(self.screen_in)
373          self.screen_in.transition_progress = 0.
374          self.screen_in.transition_state = 'in'
375          self.screen_out.transition_progress = 0.
376          self.screen_out.transition_state = 'out'
377          self.screen_in.dispatch('on_pre_enter')
378          self.screen_out.dispatch('on_pre_leave')
379  
380          self.is_active = True
381          self._anim.start(self)
382          self.dispatch('on_progress', 0)
383  
384      def stop(self):
385          '''(internal) Stops the transition. This is automatically called by the 
386          :class:`ScreenManager`. 
387          '''
388          if self._anim:
389              self._anim.cancel(self)
390              self.dispatch('on_complete')
391              self._anim = None
392          self.is_active = False
393  
394      def add_screen(self, screen):
395          '''(internal) Used to add a screen to the :class:`ScreenManager`. 
396          '''
397          self.manager.real_add_widget(screen)
398  
399      def remove_screen(self, screen):
400          '''(internal) Used to remove a screen from the :class:`ScreenManager`. 
401          '''
402          self.manager.real_remove_widget(screen)
403  
404      def on_complete(self):
405          self.remove_screen(self.screen_out)
406  
407      def on_progress(self, progression):
408          pass
409  
410      def _on_progress(self, *l):
411          progress = l[-1]
412          self.screen_in.transition_progress = progress
413          self.screen_out.transition_progress = 1. - progress
414          self.dispatch('on_progress', progress)
415  
416      def _on_complete(self, *l):
417          self.is_active = False
418          self.dispatch('on_complete')
419          self.screen_in.dispatch('on_enter')
420          self.screen_out.dispatch('on_leave')
421          self._anim = None

424  class ShaderTransition(TransitionBase):
425      '''Transition class that uses a Shader for animating the transition between 
426      2 screens. By default, this class doesn't assign any fragment/vertex 
427      shader. If you want to create your own fragment shader for the transition, 
428      you need to declare the header yourself and include the "t", "tex_in" and 
429      "tex_out" uniform:: 
430   
431          # Create your own transition. This shader implements a "fading" 
432          # transition. 
433          fs = """$HEADER 
434              uniform float t; 
435              uniform sampler2D tex_in; 
436              uniform sampler2D tex_out; 
437   
438              void main(void) { 
439                  vec4 cin = texture2D(tex_in, tex_coord0); 
440                  vec4 cout = texture2D(tex_out, tex_coord0); 
441                  gl_FragColor = mix(cout, cin, t); 
442              } 
443          """ 
444   
445          # And create your transition 
446          tr = ShaderTransition(fs=fs) 
447          sm = ScreenManager(transition=tr) 
448   
449      '''
450  
451      fs = StringProperty(None)
452      '''Fragment shader to use. 
453   
454      :attr:`fs` is a :class:`~kivy.properties.StringProperty` and defaults to 
455      None.'''
456  
457      vs = StringProperty(None)
458      '''Vertex shader to use. 
459   
460      :attr:`vs` is a :class:`~kivy.properties.StringProperty` and defaults to 
461      None.'''
462  
463      clearcolor = ColorProperty([0, 0, 0, 1])
464      '''Sets the color of Fbo ClearColor. 
465   
466      .. versionadded:: 1.9.0 
467   
468      :attr:`clearcolor` is a :class:`~kivy.properties.ColorProperty` 
469      and defaults to [0, 0, 0, 1]. 
470   
471      .. versionchanged:: 2.0.0 
472          Changed from :class:`~kivy.properties.ListProperty` to 
473          :class:`~kivy.properties.ColorProperty`. 
474      '''
475  
476      def make_screen_fbo(self, screen):
477          fbo = Fbo(size=screen.size, with_stencilbuffer=True)
478          with fbo:
479              ClearColor(*self.clearcolor)
480              ClearBuffers()
481          fbo.add(screen.canvas)
482          with fbo.before:
483              PushMatrix()
484              Translate(-screen.x, -screen.y, 0)
485          with fbo.after:
486              PopMatrix()
487          return fbo
488  
489      def on_progress(self, progress):
490          self.render_ctx['t'] = progress
491  
492      def on_complete(self):
493          self.render_ctx['t'] = 1.
494          super(ShaderTransition, self).on_complete()
495  
496      def _remove_out_canvas(self, *args):
497          if (self.screen_out and
498                  self.screen_out.canvas in self.manager.canvas.children and
499                  self.screen_out not in self.manager.children):
500              self.manager.canvas.remove(self.screen_out.canvas)
501  
502      def add_screen(self, screen):
503          self.screen_in.pos = self.screen_out.pos
504          self.screen_in.size = self.screen_out.size
505          self.manager.real_remove_widget(self.screen_out)
506          self.manager.canvas.add(self.screen_out.canvas)
507  
508          def remove_screen_out(instr):
509              Clock.schedule_once(self._remove_out_canvas, -1)
510              self.render_ctx.remove(instr)
511  
512          self.fbo_in = self.make_screen_fbo(self.screen_in)
513          self.fbo_out = self.make_screen_fbo(self.screen_out)
514          self.manager.canvas.add(self.fbo_in)
515          self.manager.canvas.add(self.fbo_out)
516  
517          self.render_ctx = RenderContext(fs=self.fs, vs=self.vs,
518                                          use_parent_modelview=True,
519                                          use_parent_projection=True)
520          with self.render_ctx:
521              BindTexture(texture=self.fbo_out.texture, index=1)
522              BindTexture(texture=self.fbo_in.texture, index=2)
523              x, y = self.screen_in.pos
524              w, h = self.fbo_in.texture.size
525              Rectangle(size=(w, h), pos=(x, y),
526                        tex_coords=self.fbo_in.texture.tex_coords)
527              Callback(remove_screen_out)
528          self.render_ctx['tex_out'] = 1
529          self.render_ctx['tex_in'] = 2
530          self.manager.canvas.add(self.render_ctx)
531  
532      def remove_screen(self, screen):
533          self.manager.canvas.remove(self.fbo_in)
534          self.manager.canvas.remove(self.fbo_out)
535          self.manager.canvas.remove(self.render_ctx)
536          self._remove_out_canvas()
537          self.manager.real_add_widget(self.screen_in)
538  
539      def stop(self):
540          self._remove_out_canvas()
541          super(ShaderTransition, self).stop()

559  class SlideTransition(TransitionBase):
560      '''Slide Transition, can be used to show a new screen from any direction: 
561      left, right, up or down. 
562      '''
563  
564      direction = OptionProperty('left', options=('left', 'right', 'up', 'down'))
565      '''Direction of the transition. 
566   
567      :attr:`direction` is an :class:`~kivy.properties.OptionProperty` and 
568      defaults to 'left'. Can be one of 'left', 'right', 'up' or 'down'. 
569      '''
570  
571      def on_progress(self, progression):
572          a = self.screen_in
573          b = self.screen_out
574          manager = self.manager
575          x, y = manager.pos
576          width, height = manager.size
577          direction = self.direction
578          al = AnimationTransition.out_quad
579          progression = al(progression)
580          if direction == 'left':
581              a.y = b.y = y
582              a.x = x + width * (1 - progression)
583              b.x = x - width * progression
584          elif direction == 'right':
585              a.y = b.y = y
586              b.x = x + width * progression
587              a.x = x - width * (1 - progression)
588          elif direction == 'down':
589              a.x = b.x = x
590              a.y = y + height * (1 - progression)
591              b.y = y - height * progression
592          elif direction == 'up':
593              a.x = b.x = x
594              b.y = y + height * progression
595              a.y = y - height * (1 - progression)
596  
597      def on_complete(self):
598          self.screen_in.pos = self.manager.pos
599          self.screen_out.pos = self.manager.pos
600          super(SlideTransition, self).on_complete()

603  class CardTransition(SlideTransition):
604      '''Card transition that looks similar to Android 4.x application drawer 
605      interface animation. 
606   
607      It supports 4 directions like SlideTransition: left, right, up and down, 
608      and two modes, pop and push. If push mode is activated, the previous 
609      screen does not move, and the new one slides in from the given direction. 
610      If the pop mode is activated, the previous screen slides out, when the new 
611      screen is already on the position of the ScreenManager. 
612   
613      .. versionadded:: 1.10 
614      '''
615  
616      mode = OptionProperty('push', options=['pop', 'push'])
617      '''Indicates if the transition should push or pop 
618      the screen on/off the ScreenManager. 
619   
620      - 'push' means the screen slides in in the given direction 
621      - 'pop' means the screen slides out in the given direction 
622   
623      :attr:`mode` is an :class:`~kivy.properties.OptionProperty` and 
624      defaults to 'push'. 
625      '''
626  
627      def start(self, manager):
628          '''(internal) Starts the transition. This is automatically 
629          called by the :class:`ScreenManager`. 
630          '''
631          super(CardTransition, self).start(manager)
632          mode = self.mode
633          a = self.screen_in
634          b = self.screen_out
635          # ensure that the correct widget is "on top"
636          if mode == 'push':
637              manager.canvas.remove(a.canvas)
638              manager.canvas.add(a.canvas)
639          elif mode == 'pop':
640              manager.canvas.remove(b.canvas)
641              manager.canvas.add(b.canvas)
642  
643      def on_progress(self, progression):
644          a = self.screen_in
645          b = self.screen_out
646          manager = self.manager
647          x, y = manager.pos
648          width, height = manager.size
649          direction = self.direction
650          mode = self.mode
651          al = AnimationTransition.out_quad
652          progression = al(progression)
653          if mode == 'push':
654              b.pos = x, y
655              if direction == 'left':
656                  a.pos = x + width * (1 - progression), y
657              elif direction == 'right':
658                  a.pos = x - width * (1 - progression), y
659              elif direction == 'down':
660                  a.pos = x, y + height * (1 - progression)
661              elif direction == 'up':
662                  a.pos = x, y - height * (1 - progression)
663          elif mode == 'pop':
664              a.pos = x, y
665              if direction == 'left':
666                  b.pos = x - width * progression, y
667              elif direction == 'right':
668                  b.pos = x + width * progression, y
669              elif direction == 'down':
670                  b.pos = x, y - height * progression
671              elif direction == 'up':
672                  b.pos = x, y + height * progression

675  class SwapTransition(TransitionBase):
676      '''Swap transition that looks like iOS transition when a new window 
677      appears on the screen. 
678      '''
679      def __init__(self, **kwargs):
680          super(SwapTransition, self).__init__(**kwargs)
681          self.scales = {}
682  
683      def start(self, manager):
684          for screen in self.screen_in, self.screen_out:
685              with screen.canvas.before:
686                  PushMatrix(group='swaptransition_scale')
687                  scale = Scale(group='swaptransition_scale')
688              with screen.canvas.after:
689                  PopMatrix(group='swaptransition_scale')
690  
691              screen.bind(center=self.update_scale)
692              self.scales[screen] = scale
693          super(SwapTransition, self).start(manager)
694  
695      def update_scale(self, screen, center):
696          self.scales[screen].origin = center
697  
698      def add_screen(self, screen):
699          self.manager.real_add_widget(screen, 1)
700  
701      def on_complete(self):
702          self.screen_in.pos = self.manager.pos
703          self.screen_out.pos = self.manager.pos
704          for screen in self.screen_in, self.screen_out:
705              for canvas in screen.canvas.before, screen.canvas.after:
706                  canvas.remove_group('swaptransition_scale')
707          super(SwapTransition, self).on_complete()
708  
709      def on_progress(self, progression):
710          a = self.screen_in
711          b = self.screen_out
712          manager = self.manager
713  
714          self.scales[b].xyz = [1. - progression * 0.7 for xyz in 'xyz']
715          self.scales[a].xyz = [0.5 + progression * 0.5 for xyz in 'xyz']
716          a.center_y = b.center_y = manager.center_y
717  
718          al = AnimationTransition.in_out_sine
719  
720          if progression < 0.5:
721              p2 = al(progression * 2)
722              width = manager.width * 0.7
723              widthb = manager.width * 0.2
724              a.x = manager.center_x + p2 * width / 2.
725              b.center_x = manager.center_x - p2 * widthb / 2.
726          else:
727              if self.screen_in is self.manager.children[-1]:
728                  self.manager.real_remove_widget(self.screen_in)
729                  self.manager.real_add_widget(self.screen_in)
730              p2 = al((progression - 0.5) * 2)
731              width = manager.width * 0.85
732              widthb = manager.width * 0.2
733              a.x = manager.x + width * (1 - p2)
734              b.center_x = manager.center_x - (1 - p2) * widthb / 2.

775  class FallOutTransition(ShaderTransition):
776      '''Transition where the new screen 'falls' from the screen centre, 
777      becoming smaller and more transparent until it disappears, and 
778      revealing the new screen behind it. Mimics the popular/standard 
779      Android transition. 
780   
781      .. versionadded:: 1.8.0 
782   
783      '''
784  
785      duration = NumericProperty(0.15)
786      '''Duration in seconds of the transition, replacing the default of 
787      :class:`TransitionBase`. 
788   
789      :class:`duration` is a :class:`~kivy.properties.NumericProperty` and 
790      defaults to .15 (= 150ms). 
791      '''
792  
793      FALLOUT_TRANSITION_FS = '''$HEADER$ 
794      uniform float t; 
795      uniform sampler2D tex_in; 
796      uniform sampler2D tex_out; 
797   
798      void main(void) { 
799          /* quantities for position and opacity calculation */ 
800          float tr = 0.5*sin(t);  /* 'real' time */ 
801          vec2 diff = (tex_coord0.st - 0.5) * (1.0/(1.0-tr)); 
802          vec2 dist = diff + 0.5; 
803          float max_dist = 1.0 - tr; 
804   
805          /* in and out colors */ 
806          vec4 cin = vec4(texture2D(tex_in, tex_coord0.st)); 
807          vec4 cout = vec4(texture2D(tex_out, dist)); 
808   
809          /* opacities for in and out textures */ 
810          float oin = clamp(1.0-cos(t), 0.0, 1.0); 
811          float oout = clamp(cos(t), 0.0, 1.0); 
812   
813          bvec2 outside_bounds = bvec2(abs(tex_coord0.s - 0.5) > 0.5*max_dist, 
814                                       abs(tex_coord0.t - 0.5) > 0.5*max_dist); 
815   
816          vec4 frag_col; 
817          if (any(outside_bounds) ){ 
818              frag_col = vec4(cin.x, cin.y, cin.z, 1.0); 
819              } 
820          else { 
821              frag_col = vec4(oout*cout.x + oin*cin.x, oout*cout.y + oin*cin.y, 
822                              oout*cout.z + oin*cin.z, 1.0); 
823              } 
824   
825          gl_FragColor = frag_col; 
826      } 
827      '''
828  
829      fs = StringProperty(FALLOUT_TRANSITION_FS)

833      '''Transition where the new screen rises from the screen centre, 
834      becoming larger and changing from transparent to opaque until it 
835      fills the screen. Mimics the popular/standard Android transition. 
836   
837      .. versionadded:: 1.8.0 
838      '''

840      duration = NumericProperty(0.2)
841      '''Duration in seconds of the transition, replacing the default of 
842      :class:`TransitionBase`. 
843   
844      :class:`duration` is a :class:`~kivy.properties.NumericProperty` and 
845      defaults to .2 (= 200ms). 
846      '''
847  
848      RISEIN_TRANSITION_FS = '''$HEADER$ 
849      uniform float t; 
850      uniform sampler2D tex_in; 
851      uniform sampler2D tex_out; 
852   
853      void main(void) { 
854          /* quantities for position and opacity calculation */ 
855          float tr = 0.5 - 0.5*sqrt(sin(t));  /* 'real' time */

857          vec2 dist = diff + 0.5; 
858          float max_dist = 1.0 - tr; 
859

862          vec4 cout = vec4(texture2D(tex_out, tex_coord0.st));

865          float oin = clamp(sin(2.0*t), 0.0, 1.0);

867   
868          bvec2 outside_bounds = bvec2(abs(tex_coord0.s - 0.5) > 0.5*max_dist, 
869                                       abs(tex_coord0.t - 0.5) > 0.5*max_dist); 
870   
871          vec4 frag_col; 
872          if (any(outside_bounds) ){ 
873              frag_col = vec4(cout.x, cout.y, cout.z, 1.0); 
874              } 
875          else { 
876              frag_col = vec4(oout*cout.x + oin*cin.x, oout*cout.y + oin*cin.y, 
877                              oout*cout.z + oin*cin.z, 1.0); 
878              } 
879   
880          gl_FragColor = frag_col; 
881      } 
882      '''

887  class ScreenManager(FloatLayout):
888      '''Screen manager. This is the main class that will control your 
889      :class:`Screen` stack and memory. 
890   
891      By default, the manager will show only one screen at a time. 
892      '''
893  
894      current = StringProperty(None, allownone=True)
895      ''' 
896      Name of the screen currently shown, or the screen to show. 
897   
898      :: 
899   
900          from kivy.uix.screenmanager import ScreenManager, Screen 
901   
902          sm = ScreenManager() 
903          sm.add_widget(Screen(name='first')) 
904          sm.add_widget(Screen(name='second')) 
905   
906          # By default, the first added screen will be shown. If you want to 
907          # show another one, just set the 'current' property. 
908          sm.current = 'second' 
909   
910      :attr:`current` is a :class:`~kivy.properties.StringProperty` and defaults 
911      to None. 
912      '''
913  
914      transition = ObjectProperty(baseclass=TransitionBase)
915      '''Transition object to use for animating the transition from the current 
916      screen to the next one being shown. 
917   
918      For example, if you want to use a :class:`WipeTransition` between 
919      slides:: 
920   
921          from kivy.uix.screenmanager import ScreenManager, Screen, 
922          WipeTransition 
923   
924          sm = ScreenManager(transition=WipeTransition()) 
925          sm.add_widget(Screen(name='first')) 
926          sm.add_widget(Screen(name='second')) 
927   
928          # by default, the first added screen will be shown. If you want to 
929          # show another one, just set the 'current' property. 
930          sm.current = 'second' 
931   
932      :attr:`transition` is an :class:`~kivy.properties.ObjectProperty` and 
933      defaults to a :class:`SlideTransition`. 
934   
935      .. versionchanged:: 1.8.0 
936   
937          Default transition has been changed from :class:`SwapTransition` to 
938          :class:`SlideTransition`. 
939      '''
940  
941      screens = ListProperty()
942      '''List of all the :class:`Screen` widgets added. You should not change 
943      this list manually. Use the 
944      :meth:`add_widget <kivy.uix.widget.Widget.add_widget>` method instead. 
945   
946      :attr:`screens` is a :class:`~kivy.properties.ListProperty` and defaults to 
947      [], read-only. 
948      '''
949  
950      current_screen = ObjectProperty(None, allownone=True)
951      '''Contains the currently displayed screen. You must not change this 
952      property manually, use :attr:`current` instead. 
953   
954      :attr:`current_screen` is an :class:`~kivy.properties.ObjectProperty` and 
955      defaults to None, read-only. 
956      '''
957  
958      def _get_screen_names(self):
959          return [s.name for s in self.screens]
960  
961      screen_names = AliasProperty(_get_screen_names, bind=('screens',))
962      '''List of the names of all the :class:`Screen` widgets added. The list 
963      is read only. 
964   
965      :attr:`screens_names` is an :class:`~kivy.properties.AliasProperty` and 
966      is read-only. It is updated if the screen list changes or the name 
967      of a screen changes. 
968      '''

970      def __init__(self, **kwargs):
971          if 'transition' not in kwargs:
972              self.transition = SlideTransition()
973          super(ScreenManager, self).__init__(**kwargs)
974          self.fbind('pos', self._update_pos)
975  
976      def _screen_name_changed(self, screen, name):
977          self.property('screen_names').dispatch(self)
978          if screen == self.current_screen:
979              self.current = name
980  
981      def add_widget(self, screen):
982          if not isinstance(screen, Screen):
983              raise ScreenManagerException(
984                  'ScreenManager accepts only Screen widget.')
985          if screen.manager:
986              if screen.manager is self:
987                  raise ScreenManagerException(
988                      'Screen already managed by this ScreenManager (are you '
989                      'calling `switch_to` when you should be setting '
990                      '`current`?)')
991              raise ScreenManagerException(
992                  'Screen already managed by another ScreenManager.')
993          screen.manager = self
994          screen.bind(name=self._screen_name_changed)
995          self.screens.append(screen)
996          if self.current is None:
997              self.current = screen.name
998  
999      def remove_widget(self, *l):
1000         screen = l[0]
1001         if not isinstance(screen, Screen):
1002             raise ScreenManagerException(
1003                 'ScreenManager uses remove_widget only for removing Screens.')
1004 
1005         if screen not in self.screens:
1006             return
1007 
1008         if self.current_screen == screen:
1009             other = next(self)
1010             if screen.name == other:
1011                 self.current = None
1012                 screen.parent.real_remove_widget(screen)
1013             else:
1014                 self.current = other
1015 
1016         screen.manager = None
1017         screen.unbind(name=self._screen_name_changed)
1018         self.screens.remove(screen)
1019

1021         if screens is None:
1022             screens = self.screens
1023 
1024         # iterate over a copy of screens, as self.remove_widget
1025         # modifies self.screens in place
1026         for screen in screens[:]:
1027             self.remove_widget(screen)

1031         parent = screen.parent
1032         if parent:
1033             parent.real_remove_widget(screen)
1034         super(ScreenManager, self).add_widget(screen)

1039     def on_current(self, instance, value):
1040         if value is None:
1041             self.transition.stop()
1042             self.current_screen = None
1043             return
1044 
1045         screen = self.get_screen(value)
1046         if screen == self.current_screen:
1047             return
1048 
1049         self.transition.stop()
1050 
1051         previous_screen = self.current_screen
1052         self.current_screen = screen
1053         if previous_screen:
1054             self.transition.screen_in = screen
1055             self.transition.screen_out = previous_screen
1056             self.transition.start(self)
1057         else:
1058             self.real_add_widget(screen)
1059             screen.pos = self.pos
1060             self.do_layout()
1061             screen.dispatch('on_pre_enter')
1062             screen.dispatch('on_enter')

1064     def get_screen(self, name):
1065         '''Return the screen widget associated with the name or raise a 
1066         :class:`ScreenManagerException` if not found. 
1067         '''
1068         matches = [s for s in self.screens if s.name == name]
1069         num_matches = len(matches)
1070         if num_matches == 0:
1071             raise ScreenManagerException('No Screen with name "%s".' % name)
1072         if num_matches > 1:
1073             Logger.warn('Multiple screens named "%s": %s' % (name, matches))
1074         return matches[0]

1113     def switch_to(self, screen, **options):
1114         '''Add a new or existing screen to the ScreenManager and switch to it. 
1115         The previous screen will be "switched away" from. `options` are the 
1116         :attr:`transition` options that will be changed before the animation 
1117         happens. 
1118  
1119         If no previous screens are available, the screen will be used as the 
1120         main one:: 
1121  
1122             sm = ScreenManager() 
1123             sm.switch_to(screen1) 
1124             # later 
1125             sm.switch_to(screen2, direction='left') 
1126             # later 
1127             sm.switch_to(screen3, direction='right', duration=1.) 
1128  
1129         If any animation is in progress, it will be stopped and replaced by 
1130         this one: you should avoid this because the animation will just look 
1131         weird. Use either :meth:`switch_to` or :attr:`current` but not both. 
1132  
1133         The `screen` name will be changed if there is any conflict with the 
1134         current screen. 
1135  
1136         .. versionadded: 1.8.0 
1137         '''
1138         assert(screen is not None)
1139 
1140         if not isinstance(screen, Screen):
1141             raise ScreenManagerException(
1142                 'ScreenManager accepts only Screen widget.')
1143 
1144         # stop any transition that might be happening already
1145         self.transition.stop()
1146 
1147         # ensure the screen name will be unique
1148         if screen not in self.screens:
1149             if self.has_screen(screen.name):
1150                 screen.name = self._generate_screen_name()
1151 
1152         # change the transition if given explicitly
1153         old_transition = self.transition
1154         specified_transition = options.pop("transition", None)
1155         if specified_transition:
1156             self.transition = specified_transition
1157 
1158         # change the transition options
1159         for key, value in iteritems(options):
1160             setattr(self.transition, key, value)
1161 
1162         # add and leave if we are set as the current screen
1163         if screen.manager is not self:
1164             self.add_widget(screen)
1165         if self.current_screen is screen:
1166             return
1167 
1168         old_current = self.current_screen
1169 
1170         def remove_old_screen(transition):
1171             if old_current in self.children:
1172                 self.remove_widget(old_current)
1173                 self.transition = old_transition
1174             transition.unbind(on_complete=remove_old_screen)
1175         self.transition.bind(on_complete=remove_old_screen)
1176 
1177         self.current = screen.name
1178 
1179     def _generate_screen_name(self):
1180         i = 0
1181         while True:
1182             name = '_screen{}'.format(i)
1183             if not self.has_screen(name):
1184                 return name
1185             i += 1
