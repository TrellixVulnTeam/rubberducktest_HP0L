11   <H1>The win32com package </H1>
12   <FONT SIZE=2><P>This document describes the win32com package in general terms.</FONT> </P>
13   <FONT SIZE=2><P>The COM support can be thought of as existing in 2 main portions - the C++ support code (the core PythonCOM module), and helper code, implemented in Python. The total package is known as "win32com".</FONT> </P>
14   <FONT SIZE=2><P>The win32com support is stand-alone. It does not require Pythonwin.</FONT> </P>
15   <H2>The win32com package </H2>
16   <FONT SIZE=2><P>To facilitate an orderly framework, the Python "ni" module has been used, and the entire package is known as "win32com". As is normal for such packages, win32com itself does not provide any functionality. Some of the modules are described below:</FONT> </P>
17   
18   <UL>
19   <B><FONT SIZE=2><LI>win32com.pythoncom - core C++ support</B>. <BR>
20   This module is rarely used directly by programmers - instead the other "helper" module are used, which themselves draw on the core pythoncom services.</FONT> </LI>
21   <B><FONT SIZE=2><LI>win32com.client package<BR>
22   </B>Support for COM clients used by Python. Some of the modules in this package allow for dynamic usage of COM clients, a module for generating .py files for certain COM servers, etc.</FONT> </LI>
23   <B><FONT SIZE=2><LI>win32com.server package<BR>
24   </B>Support for COM servers written in Python. The modules in this package provide most of the underlying framework for magically turning Python classes into COM servers, exposing the correct public methods, registering your server in the registry, etc. </LI>
25   <B><LI>win32com.axscript<BR>
26   </B>ActiveX Scripting implementation for Python.</FONT> </LI>
27   <B><FONT SIZE=2><LI>win32com.axdebug<BR>
28   </B>Active Debugging implementation for Python</FONT> </LI>
29   <B><FONT SIZE=2><LI>win32com.mapi<BR>
30   </B>Utilities for working with MAPI and the Microsoft Exchange Server</LI></UL>
31   
32   </FONT><P>&nbsp;</P>
33   <H2>The pythoncom module </H2>
34   <FONT SIZE=2><P>The pythoncom module is the underlying C++ support for all COM related objects. In general, Python programmers will not use this module directly, but use win32com helper classes and functions. </P>
35   <P>This module exposes a C++ like interface to COM - there are objects implemented in pythoncom that have methods "QueryInterface()", "Invoke()", just like the C++ API. If you are using COM in C++, you would not call a method directly, you would use pObject-&gt;Invoke( …, MethodId, argArray…). Similarly, if you are using pythoncom directly, you must also use the Invoke method to call an object's exposed method.</FONT> </P>
36   <FONT SIZE=2><P>There are some Python wrappers for hiding this raw interface, meaning you should almost never need to use the pythoncom module directly. These helpers translate a "natural" looking interface (eg, obj.SomeMethod()) into the underlying Invoke call.</P></FONT></BODY>
